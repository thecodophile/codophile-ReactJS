3. What is CDN? Why do we use it?

   -> A CDN is a content delivery network. It is a system of distributed servers that deliver content to a user based on their geographic location. The benefits of using a CDN in React Js are many.

   > CDNs can improve the performance of your website by reducing the latency of content delivery and by providing a higher level of availability.
   > CDNs can also help to improve the security of your website by providing a distributed denial of service (DDoS) protection.

4. Why is React known as React?
   
   -> React was developed for applications (Facebook) that have constantly changing data. Since React is a front-end framework or the “View” in MVC, this means that as the user clicks around and changes the app’s data, the view should “react” or change with those user events. User events being mouse clicks, typing, submitting a form.
